Project Overview

This dissertation project investigates whether Vehicle-to-Vehicle (V2V) communication can improve the efficiency of Vehicle-to-Infrastructure (V2I) data offloading in rural vehicular networks. In areas with sparse Roadside Unit (RSU) deployment, vehicles experience intermittent connectivity. By leveraging V2V communication, data can be transferred between vehicles to optimize offloading when RSUs become available.

The simulation compares scenarios with and without V2V communication using custom data offloading algorithms, testing three distinct V2V methods: RSU Detection, Directional transfer, and Load Spreading.

Key Features

  • Three V2V Methods: RSU Detection (transfers to vehicles with RSU contact), Directional (backward transfer toward RSUs), Load Spreading (balances data across vehicles)
  • Custom Simulation Environment: Rural scenarios with 2km and 6km road networks, configurable RSU placement
  • Data Loss Mitigation: Algorithm to reduce transmission failures over long distances
  • Comprehensive Metrics: Tracks offloading efficiency, contact duration, beacon messages, transfer rates, and vehicle activity
  • Python Analytics Suite: Custom scripts for range plotting, connectivity analysis, data offload tracking, and V2V transfer visualization

Research Findings

Key Conclusion: V2V communication significantly improves V2I offloading efficiency. The most effective approach combines Load Spreading (balancing data across vehicles) with Directional transfer (routing data toward RSUs). Adding data loss mitigation further increased total offloaded data by 15-20%.

The baseline scenario (V2I only) achieved 27.5% offload efficiency, while combined V2V methods reached up to 45% in sparse RSU environments.
